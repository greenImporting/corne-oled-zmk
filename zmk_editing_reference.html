core behaviors

what the & means

every binding starts with & — it references a behavior node defined in the firmware.

&kp KEYkey press — sends a keycode
most common binding
&mo LAYERmomentary layer — active while held
&lt LAYER KEYlayer-tap — tap=key, hold=layer
&mt MOD KEYmod-tap — tap=key, hold=modifier
&to LAYERto layer — switches default layer
&tog LAYERtoggle layer on/off
&sl LAYERsticky layer — next key uses layer
&sk MODsticky key — mod applies to next key only
&transtransparent — falls through to layer below
&noneblocked — does nothing, stops fall-through
&caps_wordcaps word — capitalizes until non-alpha
&key_repeatrepeats last pressed key
&bootloaderenter bootloader / flash mode
&sys_resetreset the firmware
&soft_offpower off (hold-time configurable)
&studio_unlockunlock zmk studio live editing

mod-tap &mt

tap = key, hold = modifier. useful for home row mods.

syntax

&mt MOD KEY // examples &mt LSHIFT A // tap=a, hold=lshift &mt LCTRL ESC // tap=esc, hold=lctrl &mt LGUI SPACE // tap=space, hold=lgui

modifier codes

LSHIFT / RSHIFTshift
LCTRL / RCTRLctrl
LALT / RALTalt
LGUI / RGUIgui / cmd / win
⚠ home row mods need tuning — set tapping-term-ms to avoid misfires. typical range: 175–250ms.
layer-tap &lt

syntax

&lt LAYER KEY // examples &lt 1 SPACE // tap=space, hold=layer 1 &lt 2 BSPC // tap=bspc, hold=layer 2 &lt 3 RET // tap=enter, hold=layer 3
layer numbers match the order they appear in your keymap { } block. first = 0.
tap dance &td

different action based on number of taps.

define in behaviors { }

td0: td0 { compatible = "zmk,behavior-tap-dance"; #binding-cells = <0>; tapping-term-ms = <200>; // optional bindings = <&kp LSHIFT>, <&kp CAPS>; // tap 1 = lshift, tap 2 = caps };

use in keymap

&td0 // reference by name
you can add more bindings for triple-tap, quad-tap etc.
combos

trigger a key or behavior when multiple keys are pressed together.

syntax — add inside / { }

combos { compatible = "zmk,combos"; combo_name { bindings = <&kp ESC>; key-positions = <0 1>; timeout-ms = <50>; // optional, default 50 layers = <0 1>; // optional, all layers if omitted require-prior-idle-ms = <150>; // optional }; };

key-positions

positions are numbered left-to-right, top-to-bottom starting at 0. check your board's matrix for exact numbers.

useful combo bindings

&kp ESCsend a keycode
&bootloaderenter bootloader
&soft_offpower off
&mo 1activate layer
&caps_wordcaps word
⚠ timeout-ms too high = accidental triggers. too low = hard to hit intentionally. 40–60ms is a good start.

layer structure — how to add a layer
keymap { compatible = "zmk,keymap"; default_layer { // layer 0 display-name = "base"; bindings = < &kp TAB &kp Q ... // fill all positions >; }; lower_layer { // layer 1 display-name = "lower"; bindings = < &trans &kp N1 ... // &trans = pass through >; }; my_new_layer { // layer 2 — just add another block display-name = "extra"; bindings = < &none &kp F1 ... // &none = block key entirely >; }; };
layer numbers are assigned in order of appearance — 0, 1, 2... rename the node freely, the number is positional.
global behavior settings
// put these at the top level, before / { } &lt { tapping-term-ms = <200>; // how long hold takes to activate layer. default 200 quick-tap-ms = <150>; // tap then hold = repeat key instead of layer flavor = "balanced"; // "hold-preferred" | "tap-preferred" | "balanced" | "tap-unless-interrupted" }; &mt { tapping-term-ms = <200>; flavor = "tap-preferred"; // usually best for home row mods }; &sk { release-after-ms = <1000>; // how long sticky key waits quick-release; // release on next key-up instead of next key-down }; &soft_off { hold-time-ms = <2000>; // hold duration before powering off };

quick tips
&transalways use on keys you don't want to redefine — lets the layer below through
&noneuse to intentionally block a key on a layer so nothing fires
display-nameshows on oled / zmk studio — keep it short (1–8 chars)
key countevery layer must have exactly the same number of bindings as your physical key count
buildpush to github → actions tab → wait for build → download firmware zip
flashdouble-tap reset → board appears as usb drive → drag .uf2 file onto it
